# 🎛️ PIDチューニングガイド

> リアルタイムでPIDパラメータを調整し、安定した倒立を実現する

---

## 概要

PIDチューニングは **試行錯誤のプロセス** である。
理論的な初期値から始めて、実機の挙動を見ながら調整していく。

```
初期値設定 → 1軸テスト（辺倒立）→ 3軸テスト（点倒立）→ 微調整
              ↑                      ↑
              └── ゲイン調整 ────────┘
```

---

## 1. チューニング環境の準備

### 必要なもの
- PC（Processing IDE インストール済み → [セットアップ](04_software_setup.md)）
- Bluetooth 接続（PC ↔ キューブ）
- 広くて平らな場所（テーブルの上など）
- キューブを支える手（最初は手で支えながら調整）

### Processing GUI の起動

1. `processing/tuning_gui.pde` を Processing で開く
2. COM ポート番号を自分の環境に修正
3. Run ボタンで起動
4. Bluetooth ペアリング済みのキューブの電源を入れる

## 2. PIDパラメータの役割

### パラメータ一覧

| パラメータ | 記号 | 役割 | 大きくすると | 小さくすると |
|-----------|------|------|------------|------------|
| **比例ゲイン** | Kp | 偏差に比例した修正力 | 応答が速くなるが振動しやすい | 応答が遅くなる |
| **積分ゲイン** | Ki | 定常偏差の除去 | 一方向に傾く問題が解消 | 定常偏差が残る |
| **微分ゲイン** | Kd | 振動の抑制（ダンピング） | 振動が収まるがノイズに敏感 | 振動が収まりにくい |
| **角度ゲイン** | Ka | 角度→角速度変換の感度 | 角度偏差に敏感に反応 | 鈍い反応 |
| **目標補正** | Kt | 目標角度の自動補正速度 | バランス点を速く学習 | ゆっくり学習 |

### 直感的なイメージ

```
Kp が大きすぎ: ブルブル震える（振動）
Kp が小さすぎ: ゆっくり倒れる（応答不足）

Ki が大きすぎ: 大きくうねる（オーバーシュート）
Ki が小さすぎ: 一方向に少し傾いたまま

Kd が大きすぎ: カクカクした動き（ノイズ増幅）
Kd が小さすぎ: 振動が止まらない
```

## 3. チューニング手順

### Phase 1: 1軸テスト（辺倒立）

キューブを **辺（稜線）** の上に置いて、1モーターだけで倒立させる。

> 💡 3軸同時に調整するのは難しいので、まず1軸で基本的なゲインを見つける。

#### 3.1 初期値の設定

コードのデフォルト値を出発点にする:

| パラメータ | 1号機初期値 | 2号機初期値 | まず試す値 |
|-----------|:---:|:---:|:---:|
| Kp | 60.0 | 32.0 | 30.0 |
| Ki | 12.0 | 45.0 | 30.0 |
| Kd | 90.0 | 95.0 | 90.0 |
| Ka | 5.0 | 5.5 | 5.0 |
| Kt | 1.0 | 1.3 | 1.0 |

#### 3.2 Kp の調整（最初にやる）

1. Ki = 0, Kd = 0 に設定（P制御のみ）
2. キューブを辺の上に手で支えて置く
3. Kp を少しずつ上げていく
4. **倒れる方向に軽くモーターが反応する** のを確認
5. 反応が見えたら Kp を上げて、軽く手を離してみる
6. 振動するなら Kp が高すぎ、倒れるなら低すぎ

#### 3.3 Kd の調整（次にやる）

1. Kp はそのまま、Kd を少しずつ上げる
2. 振動が抑えられていくのを確認
3. Kd が大きすぎるとカクカクした動きになる
4. **振動が収まりつつ、まだ応答がある** バランスを探す

#### 3.4 Ki の調整（最後にやる）

1. Kp, Kd はそのまま、Ki を少しずつ上げる
2. 一方向に傾く（定常偏差）が解消されるのを確認
3. Ki が大きすぎるとゆっくり大きく揺れる（積分ワインドアップ）

#### 3.5 1軸倒立の達成

- 手を離して **数秒以上** 辺の上で倒立できたらOK
- 安定しない場合は Kp/Kd の比率を変えてみる

### Phase 2: 3軸テスト（点倒立）

1軸が安定したら、角（頂点）の上に置いて3軸制御に挑戦。

#### 3.6 Roll/Pitch 調整値の設定

```
ROLL_ADJUST（rollAdj）: Roll方向のバランスオフセット
PITCH_ADJUST（pitchAdj）: Pitch方向のバランスオフセット
```

- キューブを角の上に置いて、どちらに倒れるか観察
- 倒れる方向の ADJUST 値を少し変更して重心ズレを補正
- Processing GUI のスライダーでリアルタイム調整

#### 3.7 Yaw 軸の追加（2号機のみ）

1. Roll/Pitch がある程度安定したら Kz を少しずつ上げる
2. 点倒立中に z 軸まわりの回転が抑制されるのを確認
3. yawAdj でYaw方向のトリム調整

## 4. トラブルシューティング

| 症状 | 考えられる原因 | 対処法 |
|------|--------------|--------|
| まったく反応しない | モーター配線ミス、SS信号LOW | 配線確認、`vertical` フラグ確認 |
| 激しく振動する | Kp が高すぎ | Kp を下げる、Kd を上げる |
| ゆっくり倒れる | Kp が低すぎ | Kp を上げる |
| 一方向に傾いたまま | 重心ズレ、Ki不足 | ADJUST値を調整、Ki を上げる |
| 大きくうねる | Ki が高すぎ | Ki を下げる |
| カクカクした動き | Kd が高すぎ | Kd を下げる |
| 1軸はOKだが3軸で不安定 | ミキシングの問題 | モーター方向の確認、配線の入替 |
| ホイールが一方向に加速し続ける | 積分ワインドアップ | I項にconstrain制限を確認 |
| すぐに vertical=0 になる | 閾値が厳しすぎ | コード内の角度閾値を緩める |

## 5. チューニングのコツ

### やってはいけないこと
- ❌ 複数のパラメータを同時に変える
- ❌ 大きな値をいきなり入れる
- ❌ 制御が不安定なまま手を離す

### やるべきこと
- ✅ 1つずつパラメータを変える
- ✅ 少しずつ（10〜20%刻みで）変える
- ✅ Processing GUI のグラフで角度/角速度を観察する
- ✅ 安定したパラメータはメモしておく
- ✅ 最初は手で支えながら調整する

### パラメータの保存

安定したパラメータが見つかったら、ファームウェアのデフォルト値として書き込む:

```cpp
// firmware/pico_w/3axis_control.ino の冒頭
#define _Kt 1.3    // ← ここに安定した値を書き込む
#define _Ka 5.5
#define _Kp 32.0
#define _Ki 45.0
#define _Kd 95.0
```

## 6. 重量バランスの微調整

PIDチューニングだけでは解決できない問題は、物理的な重量バランスで対処する。

### ホイールのバランス

- ホイール外周のボルト穴に M3 ボルト＋ナットを追加/除去
- 3枚のホイールの重量をなるべく均一にする
- 回転時の振動が減るよう調整

### 筐体のバランス

- バッテリーの位置を微調整（重心が角の真上に来るように）
- 配線の取り回しも重量バランスに影響する
- ROLL_ADJUST / PITCH_ADJUST で補正しきれない場合は物理調整
